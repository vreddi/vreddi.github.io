Interactive Computer Graphics
CS 418, UIUC | August 2014 - December 2014 | Champaign, Illinois, USA

These Projects was done for my Graphics class at the University of Illinois at Urbana Champaign. Each realtes to some underlying concepts in Graphics:

Project 1 : Dancing 'I'
A 2-D animation of a dancing capital letter I, which includes modeling, rendering and animation. Source creates an App with an interactive functionality for the user. The animation part of the functionality effectively makes the model 'I' dance, henceforth 'Dancing - I'. The model is a 10 Triangle model built with triangle fans effectively reducing the number of calls made to glVertex.

Project 2: Flight Simulator
Idea behind the project is to manipulate the camera to roam about the world visually duplicating the flight's cockpit view.This is a simple Flight Simulator App which provides basic interactive flight controls to the users. The only view available is the cockpit view. The aircraft can perform rolls, pitching and yaws. The aircraft translates/moves at a constant speed which cannot be changed (In the current design of the App). With the project a Camera class with a 3 coordinate system Vector and Point class with all its functionalities has been implemented.

Project 3: Would You Like Some Tea Sir?
Created a mesh-loader for this project to load the 3D teapot object from the .OBJ file. This project included perspective, hidden surfaces (via the z-buffer), texture mapping and environment mapping. Lighting was also implemented within the application to show off the reflective surface of the teapot. The reflection was created with normal vector generation for every face in the model. In the end with all the configurable mappings for the model a simple rotating (about multiple axes) animation was provided. I also used the Camera class that I built from the previous project to fit in this project for more user functionality. Now with the Camera class the user could see the teapot from different perspective/views.

Project 4: Subdivision Surfaces
This project was all about implementing the Catmull Clark Algorithm for subdividing surfaces to produce smooth corners efficiently. For the algorithm an Half Edge Data structure was implemented. Information about the edges, points and faces were stored at each subdivision in a Vector. The model used for the application was a self constructed 3D model of the letter 'I' made out of quads. With a press of a button the user could trigger to go to the next subdivision level where a face point is added to each face and each face is divided into 4 more new faces keeping information about its neighbors. All of these subdivisions involve correct calculations of their face points and edge points along with storing all the neccesary information to move to the next subdivision. Visually when the corners of the model smooth out the 'I' kinda moves on to look like a bone. Along with all this the user has the ability to toggle between mesh-view and filled-view. The User can also change color of the model with 4 - toggle switch. Lighting and texture mapping is also implemented and for the animation the model moves along a Bezier Curve whose animation speed is also configurable by the user.

Developing an Operating System
ECE 391, UIUC | January 2014 - May 2014 | Champaign, Illinois, USA

This project was for the Computer Systems Class (ECE 391). The course introduced us to the concepts and abstractions central to the development of modern computing systems, with an emphasis on the systems software that controls interaction between devices and other hardware and application programs. We covered input-output semantics, synchronization, interrupts, multitasking, virtualization of resources, protection, and resource management concepts. We were also introduced to network and storage device abstractions. In terms of practical skills, we were exposed to software development tools for source control, debugging, dependency management, and compilation, and were also focused in the context of a real operating system executing in a virtual machine. We also developed software analysis skills as well as team-based development and testing skills.

The list of topics includes:

  • x86 assembly: review of basic constructs and structures, interfacing C to assembly, macros, stack frame and calling convention
  • system software basics: resource management, virtualization, protection, system call interface, asynchronous and synchronous interactions
  • simple data structures: queues, heaps, stacks, lists
  • interrupts and exceptions: controlling generation and handling, chaining, cleanup code, interactions with device functionality, interrupt controllers
  • synchronization: primitives, memory semantics, mutual exclusion, semaphores, scheduling, race conditions
  • virtualization of the CPU: processes and scheduling
  • I/O interface: file descriptors, buffering, control operations
  • device programming: basic abstractions, character and block devices, device driver development process
  • user-level programming interfaces for file and network I/O, relationship to kernel I/O abstractions
  • virtualization of memory: hardware support and software abstractions
  • signals: semantics, generation, and delivery
  • file system abstractions and disk layout

Line Following Robot
ECE 110, UIUC| August 2013 - December 2013 | Champaign, Illinois, USA

To tie the concepts of logic gates, transistors, PWM (Pulse Width Modulator) and basic circuitry together and with the expertise gained in the labs we designed a circuit that guides a vehicle along a white piece of tape on a black table - a line-following robot. There were, of course, many difficulties to challenge us - all kinds of turns from gentle curves to 90 degree turns, zig-zags,stopping functionality,color detection and Y-splits. This lab that included the Final design challenge was very enjoyable for me and my lab partner and we saw how our circuitry and theoretical ideas translated in the practical effective motion of the vehicle.

The purpose of this design project is to make a battery powered car follow a taped track on the table. This task is achieved by circuiting a proto board connected to a certain amount of sensors that can detect the tape. The car has to conquer a tape track that consists of zig-zags, curves, right-angle turns, and splits that require a detection of color so that the car chooses the right path. Therefore the task is to try to circuit the board to conquer all the characteristics of the track in one go.

The building of the autonomous vehicle taught us a lot of things. The things that look complicated in functioning may not be that complicated under the hood. The circuitry for our vehicle was very basic yet it completed the track. We also thought about this work used in an industrial environment where such vehicles with more features are used in factories in moving stuff from one area to another autonomously.
